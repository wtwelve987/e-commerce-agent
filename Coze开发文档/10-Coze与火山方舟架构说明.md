# 十、Coze与火山方舟架构说明

## 10.1 结论

本项目建议采用：

```text
Coze：负责对话入口、知识库、工作流、用户引导和发布渠道
火山方舟：负责高质量文本、图片、视觉理解和视频生成
```

不要把 Coze 自带模型当作唯一的核心生成模型。Coze 更适合作为业务编排层；真正决定生成质量的文本、图片、视频能力，建议通过火山方舟 API 接入。

## 10.2 为什么不只用 Coze 模型

Coze 的优势是搭建智能体、配置知识库、串联工作流和发布应用。但在电商商图视频场景中，单靠 Coze 自带模型容易出现以下问题：

- 商品图 Prompt 不够稳定，容易泛化
- 平台风格适配不够精细
- 图片、视频生成质量受平台内置能力限制
- 多步骤任务中容易跳过知识库或规则判断
- 对主图合规、平台尺寸、素材复用等业务规则执行不够稳定

因此，Coze 适合做“应用壳”和“流程控制”，不适合单独承担全部核心创作能力。

## 10.3 推荐能力分工

| 功能模块 | 推荐实现方式 | 说明 |
|----------|--------------|------|
| 用户对话入口 | Coze Bot | 接收用户需求，识别意图，引导补充信息 |
| 人设与回复逻辑 | Coze Bot | 约束流程、回复风格、知识库调用规则 |
| 知识库召回 | Coze 知识库 | 召回平台规范、关键词、模板、文案规则 |
| 快速生成流程 | Coze 工作流 | 固定步骤：平台、品类、用途、商品信息、是否生成视频 |
| 自定义生成流程 | Coze 工作流 | 支持用户逐步控制图片、视频、处理、收藏 |
| 平台规范判断 | Coze 工作流 + 知识库 | 判断尺寸、比例、主图限制、水印限制 |
| 标题/卖点生成 | 火山方舟文本模型 | 根据知识库结果生成更稳定的电商文案 |
| 图片 Prompt 生成 | 火山方舟文本模型 | 把商品信息、平台规范、模板组合成高质量 Prompt |
| 商品图片生成 | 火山方舟图片生成模型 | 生成白底图、场景图、模特图、促销图等 |
| 商品图理解 | 火山方舟视觉理解模型 | 分析用户上传商品图，提取主体、颜色、材质和卖点 |
| 商品视频生成 | 火山方舟视频生成模型 | 根据图片或脚本生成营销短视频 |
| 素材收藏/查询 | Coze 工作流 + 知识库/数据库 | 收藏模板和 Prompt，按 user_id 做隔离 |

## 10.4 工作流推荐结构

图片生成工作流建议：

```text
用户输入
→ 识别平台、品类、商品名称、用途
→ 调用知识库 RecallKnowledge
→ 获取平台规范、图片模板、关键词
→ 判断是否为主图
→ 组装图片 Prompt
→ 调用火山方舟图片生成 API
→ 返回图片和说明
```

视频生成工作流建议：

```text
用户输入或已生成图片
→ 调用知识库 RecallKnowledge
→ 获取平台视频规范和脚本模板
→ 生成视频脚本
→ 调用火山方舟视频生成 API
→ 返回视频和脚本摘要
```

用户上传图片处理工作流建议：

```text
用户上传图片
→ 读取 sys_images
→ 调用火山方舟视觉理解模型分析图片
→ 提取商品主体、品类、颜色、卖点
→ 调用知识库获取平台规范
→ 生成优化 Prompt 或处理指令
→ 调用图片生成/编辑能力
```

## 10.5 Coze中需要保留的配置

即使核心生成接入火山方舟，Coze 中仍然需要配置：

- Bot 人设与回复逻辑
- 开场白
- 快捷指令
- 5 个知识库
- 知识库按需调用规则
- 变量 `sys_images`
- 快速生成、自定义生成、关键词补全、素材收藏等工作流
- 发布渠道

Coze 的重点是把用户需求变成稳定的结构化参数，再把参数传给火山方舟。

## 10.6 火山方舟接入位置

推荐在 Coze 工作流中通过以下方式接入火山方舟：

1. 创建「HTTP 请求」节点或自定义插件
2. 配置火山方舟 API 地址
3. 在请求头中配置鉴权信息
4. 将平台、品类、商品信息、知识库召回结果作为请求参数
5. 接收火山方舟返回的文本、图片或视频结果
6. 在 Coze 中格式化输出给用户

火山方舟官方文档：

- [火山方舟文档](https://www.volcengine.com/docs/82379)

## 10.7 阶段性落地建议

如果先做 Demo：

- 可以先用 Coze 自带模型跑通流程
- 重点验证知识库、工作流、变量和用户路径是否正确
- 图片和视频质量先不作为最终标准

如果准备做正式可用版本：

- 标题、卖点、Prompt 生成接火山方舟文本模型
- 图片生成接火山方舟图片生成模型
- 图片理解接火山方舟视觉理解模型
- 视频生成接火山方舟视频生成模型
- Coze 只保留编排、召回、变量和渠道能力

## 10.8 核心原则

```text
Coze 负责把流程跑稳。
火山方舟负责把内容生成好。
知识库负责把业务规则说清楚。
工作流负责把每一步固定下来。
```
