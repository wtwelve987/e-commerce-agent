# 四、工作流详细配置

## 4.0 本章先读：工作流在哪里配、怎么接入主Bot

工作流不是在「资源库」里上传文档，也不是单独给用户看的页面。工作流是 Bot 的“技能/工具”，需要在 Bot 编排页里创建，并挂到主 Bot 上让 Bot 调用。

### 4.0.1 工作流创建入口

在 Coze 页面中按下面路径操作：

```text
项目管理
→ 打开「电商AI商图视频助手」Bot/项目
→ 进入 Bot 编排/配置页面
→ 左侧「技能」
→ 展开「工作流」
→ 点击「+」
→ 新建工作流
```

如果你的页面中有「资源库 → 工作流」入口，也可以在那里创建可复用工作流；但创建后仍然需要回到 Bot 编排页，在「技能 → 工作流」里把它添加到当前 Bot，否则主 Bot 不会调用它。

### 4.0.2 需要创建几个工作流

本项目建议创建 10 个工作流，每个工作流负责一个明确动作：

| 序号 | 工作流名称 | 主用途 | 主Bot调用方式 |
|------|------------|--------|---------------|
| 1 | 快速生成 | 平台、品类、风格、商品信息一键生成图片/视频 | `/快速` 或用户说“快速生成” |
| 2 | 图片生成 | 根据商品描述生成商品图 | 用户说“生成图片/做主图/做场景图” |
| 3 | 视频生成 | 根据图片和脚本生成营销视频 | 用户说“做视频/图片转视频” |
| 4 | 关键词补全 | 查询平台和品类关键词 | `/关键词` 或用户要求优化关键词 |
| 5 | 模板收藏 | 收藏当前 Prompt、模板或素材记录 | `/收藏` |
| 6 | 素材查询 | 查询用户历史素材 | `/我的素材` |
| 7 | 去背景 | 去除用户上传图片背景 | `/去背景` |
| 8 | 图片编辑 | 换背景、调尺寸、加水印等 | `/换背景` 或用户要求处理图片 |
| 9 | 平台切换 | 用同一商品生成另一个平台版本 | `/换平台` |
| 10 | 链接提取 | 从商品链接提取商品信息 | `/提取 + 链接` |

### 4.0.3 主Bot怎么接入工作流

每个工作流创建完成后，都要做三步接入：

1. 在 Bot 编排页左侧「技能 → 工作流」中确认该工作流已添加到当前 Bot
2. 在「快捷指令」中把相关指令绑定到对应工作流
3. 在「人设与回复逻辑」中写明什么场景调用哪个工作流

快捷指令绑定关系：

| 快捷指令 | 绑定工作流 |
|----------|------------|
| `/快速` `/quick` | 快速生成 |
| `/关键词` `/keywords` | 关键词补全 |
| `/我的素材` `/assets` | 素材查询 |
| `/收藏` `/save` | 模板收藏 |
| `/去背景` `/removebg` | 去背景 |
| `/换背景` `/bg` | 图片编辑 |
| `/换平台` `/switch` | 平台切换 |
| `/提取` `/extract` | 链接提取 |

`/自定义` 可以不绑定单一工作流，由主 Bot 通过对话引导用户分步调用「图片生成」「图片编辑」「视频生成」「模板收藏」等工作流。

### 4.0.4 人设中需要说明的工作流调用规则

在 `01-Bot基础配置.md` 的人设中已经配置了知识库调用规则。这里还需要在同一段人设里补充工作流调用规则，建议加入：

```text
当用户明确说“快速生成”或输入 /快速 时，调用「快速生成」工作流。
当用户要求生成商品图片、主图、场景图、模特图、促销图或详情图时，调用「图片生成」工作流。
当用户要求生成视频、营销短视频、图片转视频时，调用「视频生成」工作流。
当用户要求补充关键词、优化标题关键词、查询平台热词时，调用「关键词补全」工作流。
当用户要求收藏当前模板、收藏当前素材、保存 Prompt 时，调用「模板收藏」工作流。
当用户输入 /我的素材 或要求查看历史素材时，调用「素材查询」工作流。
当用户要求去背景时，调用「去背景」工作流。
当用户要求换背景、调尺寸、加水印时，调用「图片编辑」工作流。
当用户要求换成另一个平台版本时，调用「平台切换」工作流。
当用户粘贴商品链接或输入 /提取 + 链接时，调用「链接提取」工作流。
```

### 4.0.5 工作流输入输出怎么和主Bot连接

每个工作流都有 Start 节点和 End 节点：

- Start 节点：定义主 Bot 传入工作流的参数，例如 `platform`、`category`、`product_name`、`image_url`
- End 节点：定义工作流返回给主 Bot 的结果，例如 `image_urls`、`video_url`、`prompt`、`script_content`

主 Bot 调用工作流时，需要把对话中收集到的信息映射到 Start 节点输入变量。工作流结束后，主 Bot 使用 End 节点输出组织回复。

示例：

```text
用户：帮我做一张拼多多白色运动鞋白底主图

主Bot识别：
platform = 拼多多
category = 服饰
image_style = 白底主图
product_name = 白色运动鞋

调用：
图片生成工作流

工作流返回：
image_urls
prompt
recommended_keywords

主Bot回复：
展示图片，并询问是否继续做视频、去背景、收藏或换平台。
```

### 4.0.6 配置顺序

建议按下面顺序配置，不要一上来就配置全部复杂节点：

1. 先创建 10 个空工作流，只配置名称、描述、Start/End 变量
2. 把 10 个工作流添加到主 Bot 的「技能 → 工作流」
3. 在「快捷指令」中绑定对应工作流
4. 先跑通「关键词补全」这种简单工作流
5. 再配置「图片生成」「视频生成」这种调用外部模型/API 的工作流
6. 最后配置收藏、素材查询、平台切换、链接提取

## 4.0.7 工作流中模型与插件的选择

工作流里可以直接配置模型节点，也可以配置插件节点或 HTTP 请求节点。

不一定必须使用 `03-插件配置.md` 中的插件。插件只是 Coze 插件市场提供的快捷能力，适合 Demo 和快速验证；正式版本建议把核心生成能力接到火山方舟。

推荐规则：

| 节点类型 | 适合做什么 | 推荐用法 |
|----------|------------|----------|
| LLM 节点 | 意图识别、Prompt 生成、文案生成、结构化整理 | Demo 可用 Coze 内置模型；正式版本可改为 HTTP 请求火山方舟文本模型 |
| Knowledge 节点 | 检索平台规范、模板、关键词、文案规则 | 继续使用 Coze 知识库 |
| Plugin 节点 | 调用 Coze 插件市场能力 | 快速验证时使用，如通义万相、可灵、remove.bg |
| HTTP 请求节点 | 调用外部模型或业务接口 | 正式版本推荐用于接火山方舟图片、视频、视觉理解、文本模型 |
| Condition 节点 | 判断分支 | 用于判断平台、用途、是否主图、是否生成视频 |

因此，后文节点表中写到 `Plugin` 的地方，可以按实际方案替换：

| 文档中的节点 | Demo方案 | 正式方案 |
|--------------|----------|----------|
| 图片生成 Plugin | 通义万相/DALL·E 3 插件 | HTTP 请求火山方舟图片生成 API |
| 视频生成 Plugin | 可灵/Vidu/Seedance 插件 | HTTP 请求火山方舟视频生成 API |
| 去背景 Plugin | remove.bg 插件 | HTTP 请求图片处理 API |
| 网页抓取 Plugin | 网页读取插件 | HTTP 请求自建抓取服务或第三方接口 |
| LLM生成Prompt | Coze 内置模型 | HTTP 请求火山方舟文本模型 |

如果要追求效果，优先把“生成内容质量”的节点接火山方舟；Coze 保留流程编排、知识库、变量和分支判断。

## 4.1 快速生成工作流

**工作流名称**：`快速生成`

**触发方式**：Bot判断走快速流程时调用，或 `/快速` 快捷指令触发

### 节点配置

| 节点编号 | 节点名称 | 节点类型 | 详细配置 |
|---------|---------|---------|---------|
| 1 | 开始 | Start | 定义输入变量（见下方变量表） |
| 2 | 知识库检索 | Knowledge | 检索知识库：`预置模板库`；检索词：`platform + category + image_style` |
| 3 | LLM生成Prompt | LLM | 模型：豆包·1.6·lite；系统提示词见下方 |
| 4 | 图片生成 | Plugin | 插件：通义万相；参数映射见下方 |
| 5 | 是否生成视频 | Condition | 条件：`video_style` 为空 → 跳到节点7；非空 → 继续节点6 |
| 6 | 视频生成 | Plugin | 插件：可灵；参数映射见下方 |
| 7 | 结束 | End | 定义输出变量（见下方变量表） |

### 变量定义

**Start 节点输入变量**：

| 变量名 | 类型 | 必填 | 默认值 | 说明 |
|--------|------|------|--------|------|
| `user_id` | String | 是 | - | 当前登录用户ID，用于素材隔离 |
| `platform` | String | 是 | `拼多多` | 目标平台：拼多多/淘宝/京东/抖音/快手/小红书 |
| `category` | String | 是 | - | 品类：服饰/美妆/食品/家居/数码/母婴/其他 |
| `image_style` | String | 是 | `白底主图` | 图片风格：白底主图/场景图/模特图/促销图/详情图 |
| `product_name` | String | 是 | - | 商品名称 |
| `product_info` | String | 否 | - | 商品卖点描述 |
| `video_style` | String | 否 | 空 | 视频模板：展示型/对比型/开箱型/促销型，为空则不生成视频 |

**End 节点输出变量**：

| 变量名 | 类型 | 说明 |
|--------|------|------|
| `image_urls` | Array\<String\> | 生成的图片URL列表 |
| `video_url` | String | 生成的视频URL（可选） |
| `prompt` | String | 使用的图片Prompt |
| `platform_info` | String | 平台信息（名称+尺寸） |

### 节点3 — LLM系统提示词

```
你是电商图片Prompt工程师。根据输入信息生成图片Prompt。

输入：
- platform：目标平台
- category：品类
- image_style：图片风格
- product_name：商品名称
- product_info：卖点
- template_result：知识库检索到的模板

按平台确定尺寸和风格：
- 拼多多：750×750，简洁直白，价格导向
- 淘宝：800×800，品质感，场景丰富
- 京东：800×800，专业严谨，细节突出
- 抖音：1080×1080，视觉冲击，年轻潮流
- 快手：750×750，真实接地气
- 小红书：1080×1440，种草美感，生活化

如果有匹配的模板，基于模板修改商品名称和卖点；如果没有，根据平台规范从头生成。

输出JSON：
{"prompt": "英文图片描述", "negative_prompt": "排除元素", "size": "宽*高", "platform": "平台名"}
```

### 节点4 — 图片生成插件参数映射

| 插件参数 | 映射来源 |
|---------|---------|
| `prompt` | 节点3输出的 `prompt` |
| `negative_prompt` | 节点3输出的 `negative_prompt` |
| `size` | 节点3输出的 `size` |
| `n` | 固定值 `3` |

### 节点6 — 视频生成插件参数映射

| 插件参数 | 映射来源 |
|---------|---------|
| `image_url` | 节点4输出的第一张图片URL |
| `prompt` | 按platform+video_style拼接："E-commerce {video_style} video for {product_name}, {platform} style" |
| `duration` | `5` |
| `aspect_ratio` | 按platform映射：拼多多→`1:1`，淘宝→`3:4`，抖音→`9:16`，快手→`9:16`，小红书→`3:4`，京东→`16:9` |

### 节点5 — Condition 条件

| 条件 | 判断 | 跳转 |
|------|------|------|
| `video_style` == 空 | True | → 节点7（结束） |
| `video_style` != 空 | False | → 节点6（视频生成） |

## 4.2 图片生成工作流

**工作流名称**：`图片生成`

**触发方式**：自定义生成节点2，或Bot识别用户要生图

| 节点编号 | 节点名称 | 节点类型 | 详细配置 |

| 2 | 关键词检索 | Knowledge | 检索知识库：`商品关键词库`；检索词：`platform + category` |
| 3 | 是否使用模板 | Condition | `use_template` == true → 节点4a；false → 节点4b |
| 4a | 模板检索 | Knowledge | 检索知识库：`预置模板库`；检索词：`platform + image_style` |
| 4b | LLM优化Prompt | LLM | 模型：豆包·1.6·lite；系统提示词见下方 |
| 5 | 图片生成 | Plugin | 插件：通义万相；参数映射见下方 |
| 6 | 结束 | End | 定义输出变量 |

### 变量定义

**Start 节点输入变量**：

| 变量名 | 类型 | 必填 | 默认值 | 说明 |
|--------|------|------|--------|------|
| `user_id` | String | 是 | - | 当前登录用户ID，用于素材隔离 |
| `product_desc` | String | 是 | - | 商品描述（自由文本） |
| `category` | String | 是 | - | 品类 |
| `platform` | String | 是 | `拼多多` | 目标平台 |

```
{"prompt": "...", "negative_prompt": "...", "size": "宽*高", "platform": "平台名"}
```

### 节点5 — 图片生成插件参数映射

| 节点编号 | 节点名称 | 节点类型 | 详细配置 |
|---------|---------|---------|---------|

| 3a | LLM生成脚本 | LLM | 模型：豆包·1.6·lite；系统提示词见下方 |

| 5 | 结束 | End | 定义输出变量 |

### 变量定义

**Start 节点输入变量**：

| 变量名 | 类型 | 必填 | 默认值 | 说明 |
|--------|------|------|--------|------|
| `user_id` | String | 是 | - | 当前登录用户ID，用于素材隔离 |
| `image_url` | String | 是 | - | 商品图片URL |

| `platform` | String | 是 | `拼多多` | 目标平台 |
| `video_desc` | String | 否 | 空 | 自定义视频描述 |
| `video_template` | String | 否 | 空 | 视频模板类型：展示型/对比型/开箱型/促销型 |

**End 节点输出变量**：

| 变量名 | 类型 | 说明 |
|--------|------|------|
| `video_url` | String | 生成的视频URL |

### 节点3a — LLM系统提示词

```
你是电商短视频脚本策划师。按平台风格生成视频脚本。

输入：
- image_url：商品图片
- product_name：商品名称
- platform：目标平台
- video_desc：用户自定义视频描述

按平台调整：
- 拼多多：直白卖货，价格突出，节奏快
- 淘宝：品质展示，细节丰富，节奏适中

输出JSON：
{
  "scenes": [{"time": "0-3s", "animation": "zoom_in", "text": "字幕", "transition": "fade"}],
  "video_prompt": "英文视频生成描述",

```

| `duration` | 节点3a输出的 `total_duration` 对应值，默认 `5` |

## 4.4 关键词补全工作流

**工作流名称**：`关键词补全`

**触发方式**：`/关键词` 快捷指令，或生图时自动触发

### 节点配置

| 节点编号 | 节点名称 | 节点类型 | 详细配置 |

## 4.8 图片编辑工作流

**工作流名称**：`图片编辑`

**触发方式**：`/换背景` 快捷指令，或自定义生成节点3

### 节点配置

| 节点编号 | 节点名称 | 节点类型 | 详细配置 |
|---------|---------|---------|---------|
| 1 | 开始 | Start | 定义输入变量 |
| 2 | 图片处理 | Code | Python代码按action处理（代码见下方） |
| 3 | 结束 | End | 定义输出变量 |

### 变量定义

**Start 节点输入变量**：

| 变量名 | 类型 | 必填 | 默认值 | 说明 |
|--------|------|------|--------|------|
| `user_id` | String | 是 | - | 当前登录用户ID，用于素材隔离 |
| `image_url` | String | 是 | - | 原始图片URL |
| `action` | String | 是 | - | 操作类型：换背景/调尺寸/加水印 |
| `params` | String | 否 | 空 | 操作参数（颜色/尺寸/水印文字） |
| `platform` | String | 否 | `拼多多` | 平台（调尺寸时按平台规范） |

**End 节点输出变量**：

| 变量名 | 类型 | 说明 |

### 节点2 — Code 代码

```python
def main(image_url: str, action: str, params: str, platform: str) -> dict:
    size_map = {
        "拼多多": "750*750",
        "淘宝": "800*800",
        "京东": "800*800",
        "抖音": "1080*1080",
        "快手": "750*750",
        "小红书": "1080*1440"
    }

    if action == "调尺寸":
        target_size = params if params else size_map.get(platform, "750*750")
        return {
            "processed_image_url": image_url,
            "message": f"已生成调尺寸参数：{target_size}"
        }

    if action == "换背景":
        color = params if params else "白色"
        return {
            "processed_image_url": image_url,
            "message": f"已生成换背景参数：{color}"
        }

    if action == "加水印":
        text = params if params else "店铺名"
        return {
            "processed_image_url": image_url,
            "message": f"已生成水印参数：{text}"
        }

    return {
        "processed_image_url": image_url,
        "message": "未识别操作类型"
    }
```

> **注意**：Coze工作流的 Code 节点更适合做参数整理，不适合直接承载真实图片编辑。实际实现建议：
> - **换背景**：先去背景，再走专门的图片合成/重绘能力
> - **调尺寸**：在图片生成或图片处理插件里直接生成目标尺寸
> - **加水印**：优先使用专门的图片处理 API；如果平台限制较多，再降级为提示用户在外部工具处理
