# 二、知识库配置

## 2.1 创建5个知识库

在新版 Coze 左侧点击「资源库」→「知识库」→「+ 创建」或「+ 添加知识库」，逐个创建：

---

### 知识库1：商品关键词库

| 配置项 | 值 | 说明 |
|-------|-----|------|
| 知识库名称 | `商品关键词库` | 知识库显示名称 |
| **文档解析策略** | **精准解析** | 提取图片、表格、OCR等元素（如纯文本可选"快速解析"） |
| └ 提取内容 | ☑图片元素 ☑扫描件(OCR) ☑表格元素 | 根据文档类型勾选，纯文本可都不选 |
| **分段策略** | **按层级分段** | 按文档层级结构分段，保留标题层级信息 |
| └ 分段层级 | `3` | 按前3级标题（# ## ###）切分，保持结构完整 |
| └ 检索切片保留层级信息 | ☑勾选 | 召回时保留标题层级，便于定位内容位置 |
| **配置存储** | **平台共享存储** | 使用Coze平台默认存储（如需数据隔离可选"云搜索服务"） |

**上传文件**（按以下内容创建 .md 文件并上传）：

## 2.2 知识库关联Bot

在「项目管理」中打开对应Bot/项目，在知识库或资源引用配置区，将上述5个知识库全部关联到Bot。每个知识库的调用策略选择「按需调用」。

### 2.2.1 Coze页面操作位置

在 Bot 编排页面左侧配置区操作：

1. 进入「项目管理」或 Bot 编辑页
2. 打开对应 Bot/项目
3. 在左侧找到「知识」模块
4. 点击「文本」
5. 点击知识库名称右侧的下拉或添加按钮
6. 勾选并关联以下5个知识库：
   - `商品关键词库`
   - `预置模板库`
   - `平台规范库`
   - `文案模板库`
   - `我的素材库`

如果页面中出现「知识库设置」弹窗，按下面配置：

| 设置项 | 建议值 |
|-------|--------|
| 调用方式 | 按需调用 |
| 搜索策略 | 混合 |
| 最大召回数量 | 5 |
| 最小匹配度 | 0.60 |
| 查询改写 | 开启 |
| 结果重排 | 开启 |
| 显示来源 | 可关闭 |

### 2.2.2 按需调用需要写入人设与回复逻辑

选择「按需调用」后，Bot 不会自动在所有问题里检索知识库，需要在「人设与回复逻辑」中明确要求调用知识库。

建议在 Bot 的「人设与回复逻辑」里加入：

```text
当用户要求生成商品图片、商品视频、商品标题、卖点文案、平台素材或上传平台内容时，必须按需调用知识库 RecallKnowledge。

需要优先检索：
1. 平台规范库：确认平台尺寸、比例、风格和合规要求
2. 预置模板库：获取图片模板或视频脚本模板
3. 商品关键词库：获取平台和品类关键词
4. 文案模板库：生成标题、卖点、字幕时调用
5. 我的素材库：仅查询当前 user_id 对应的素材记录

生成主图前必须判断平台和使用位置。如果是平台主图，不得添加水印、文字、logo。
```

### 2.2.3 图片生成工作流中的应用位置

如果图片生成是通过工作流完成，需要在工作流里使用知识库，推荐放在 Prompt 生成节点之前：

1. 用户输入节点：接收平台、品类、商品名称、生成用途等信息
2. 知识库召回节点：按平台、品类、用途检索知识库
3. 规则判断节点：判断是否为主图、详情图、视频封面、内部草稿等场景
4. Prompt 组装节点：把知识库结果合并成图片生成 Prompt
5. 图片生成节点：调用图片生成模型
6. 结果输出节点：返回图片和对应说明

知识库召回建议按场景拆开：

| 场景 | 优先调用知识库 |
|------|----------------|
| 生成商品图片 | 平台规范库、预置模板库、商品关键词库 |
| 生成商品视频 | 平台规范库、预置模板库、文案模板库 |
| 生成标题卖点 | 商品关键词库、文案模板库 |
| 查询收藏素材 | 我的素材库 |

## 2.3 素材库隔离规则

- `我的素材库` 必须按 `user_id` 做用户隔离，不能做全局共享库
- 收藏工作流写入时，要把 `user_id` 一并写入记录，便于后续检索和过滤
- 素材查询时，检索条件必须默认带上当前用户的 `user_id`
- 如果后续增加团队协作，再单独新增团队素材库，不要复用个人素材库

### 2.3.1 素材库隔离操作位置

`user_id` 隔离不是在知识库设置弹窗里完成，而是在素材收藏和素材查询工作流中完成。

收藏素材工作流中处理：

1. 在用户触发收藏时，获取当前用户的 `user_id`
2. 写入素材记录时，把 `user_id` 一起写入 `我的素材库`
3. 每条素材记录都必须包含 `user_id`

查询素材工作流中处理：

1. 在查询前获取当前用户的 `user_id`
2. 检索 `我的素材库` 时，把 `user_id` 作为过滤条件或检索关键词
3. 返回结果前再次确认素材记录中的 `user_id` 与当前用户一致

建议素材记录格式：

```markdown
---
用户ID：{user_id}
名称：{template_name}
类型：{type}
平台：{platform}
品类：{category}
Prompt：{prompt}
创建时间：{timestamp}
备注：{notes}
---
```

## 2.4 合规提醒

- 主图规范里明确写了“主图不可有水印、文字、logo”
- 加水印只适用于店铺素材、详情图、内部草稿或视频封面等允许场景
- 生成前最好先判断平台和使用位置，避免一键生成后无法上架

### 2.4.1 合规规则操作位置

合规规则建议同时放在两个位置：

1. Bot「人设与回复逻辑」
2. 图片生成工作流的 Prompt 组装节点之前

在图片生成工作流中，建议增加一个「用途判断」或「规则判断」节点：

| 使用位置 | 是否允许文字/水印/logo |
|----------|------------------------|
| 平台主图 | 不允许 |
| 白底主图 | 不允许 |
| 详情图 | 可按需要允许 |
| 店铺素材 | 可按需要允许 |
| 视频封面 | 可按需要允许 |
| 内部草稿 | 可按需要允许 |

建议判断规则：

```text
如果 use_scene 为 主图 或 白底主图：
禁止在图片 Prompt 中加入文字、水印、logo、价格标签、促销字样。

如果 use_scene 为 详情图、店铺素材、视频封面、内部草稿：
可以根据用户需求加入文字、水印或品牌元素。
```
