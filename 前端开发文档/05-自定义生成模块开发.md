# 05 — 自定义生成模块开发

## 一、模块概述

自定义生成是节点式操作面板，用户逐步控制4个节点：图片生成→图片处理→视频生成→保存收藏。每个节点独立操作，完成后解锁下一个。

```
StepProgress: ①图片生成 → ②图片处理 → ③视频生成 → ④保存收藏
               ●当前       ○可选       ○可选       ○可选
```

与快速生成的区别：
- 快速生成：线性4步表单，一键出结果
- 自定义生成：每个节点可反复操作，精细化控制

## 二、4个节点详细设计

### 节点1：ImageGenNode 图片生成

#### 布局

```
┌────────────────────────────────────────────────────────┐
│  📸 图片生成                                            │
│                                                        │
│  输入方式: ○ 自由描述  ● 选择模板                       │
│                                                        │
│  ── 自由描述模式 ──                                     │
│  商品描述: [________________________________]           │
│  💡推荐关键词: [白底] [8K] [细节] [高清] [点击采纳→]     │
│  已采纳: [白底 ×] [8K ×]                               │
│                                                        │
│  ── 选择模板模式 ──                                     │
│  ┌─────┐ ┌─────┐ ┌─────┐                              │
│  │模板A│ │模板B│ │模板C│  ← 按平台+品类过滤             │
│  └─────┘ └─────┘ └─────┘                              │
│                                                        │
│  [生成图片]                                            │
│                                                        │
│  ── 生成结果 ──                                        │
│  ┌──────┐ ┌──────┐ ┌──────┐                           │
│  │ 图1  │ │ 图2✓ │ │ 图3  │                           │
│  └──────┘ └──────┘ └──────┘                           │
│                                                        │
│  选中后 → [✓ 确认选择，进入下一步]                      │
└────────────────────────────────────────────────────────┘
```

#### 两种输入模式

| 模式 | 输入 | 说明 |
|------|------|------|
| 自由描述 | 文本输入框 + KeywordTags | 用户自由描述商品，可补充关键词 |
| 选择模板 | TemplateCard 列表 | 从预置模板中选择，按平台+品类过滤 |

#### 工作流调用

```javascript
async function generateImage() {
  const params = {
    user_id: coze.getUserId(),
    product_desc: state.productInfo.name + ' ' + state.productInfo.sellingPoints.join('、'),
    category: state.productInfo.category,
    platform: state.currentPlatform,
    keywords: state.productInfo.keywords.join(','),
    use_template: isTemplateMode ? selectedTemplateId : '',
  };

  showSkeleton('image');

  try {
    const result = await coze.runWorkflow({
      workflowId: IMAGE_GEN_WORKFLOW_ID,
      parameters: params,
    });

    state.currentResult.images = result.images;
    state.currentResult.prompt = result.prompt;

    // 如果工作流返回推荐关键词
    if (result.recommend_keywords) {
      updateKeywordRecommendations(result.recommend_keywords);
    }
  } catch (error) {
    showToast('error', '图片生成失败，请重试');
  } finally {
    hideSkeleton();
  }
}
```

#### 确认选择

用户点击"确认选择"后：
1. 记录 `currentResult.selectedImageIndex`
2. 标记 `customProgress.imageGenCompleted = true`
3. 解锁节点2
4. `customProgress.currentNode = 2`

```
└────────────────────────────────────────────────────────┘
```

#### 4种操作

| 操作 | 工作流 | 参数 | 结果 |
|------|--------|------|------|
| 去背景 | 去背景工作流 | image_url | processed_image_url |
| 换背景 | 图片编辑工作流 | image_url, action='换背景', params={color} | processed_image_url |
| 调尺寸 | 图片编辑工作流 | image_url, action='调尺寸', params={width, height} | processed_image_url |
| 加水印 | 图片编辑工作流 | image_url, action='加水印', params={text, position} | processed_image_url |

#### 可多次处理

- 每次处理基于上次结果（processedImageUrl → 再次处理）
- 维护处理历史栈，支持撤销
- 跳过则直接进入节点3

```javascript
const editHistory = []; // 处理历史栈

async function processImage(action, params) {
  const imageUrl = editHistory.length > 0
    ? editHistory[editHistory.length - 1]
    : state.currentResult.images[state.currentResult.selectedImageIndex].url;

  const workflowId = action === '去背景'
    ? REMOVE_BG_WORKFLOW_ID
    : IMAGE_EDIT_WORKFLOW_ID;

  const result = await coze.runWorkflow({
    workflowId,
    parameters: {
      user_id: coze.getUserId(),
      image_url: imageUrl,
      action,
      params: JSON.stringify(params),
      platform: state.currentPlatform,
    },
  });

  editHistory.push(result.processed_image_url);
  state.currentResult.processedImageUrl = result.processed_image_url;
}

function undoEdit() {
  if (editHistory.length > 0) {
    editHistory.pop();
    state.currentResult.processedImageUrl = editHistory.length > 0
      ? editHistory[editHistory.length - 1]
      : null;
  }
}
```

```javascript
async function generateVideo() {
```

## 三、节点进度管理

```javascript
const customProgress = {
  currentNode: 1,
  completedNodes: [],
  imageGenCompleted: false,
  imageEditCompleted: false,
  videoGenCompleted: false,
};

// 节点解锁规则
function canAccessNode(node) {
  switch (node) {
    case 1: return true;                          // 始终可访问
    case 2: return customProgress.imageGenCompleted;
    case 3: return customProgress.imageGenCompleted;
    case 4: return customProgress.imageGenCompleted;
  }
}

// 节点完成标记
function completeNode(node) {
  customProgress.completedNodes.push(node);
  if (node < 4) customProgress.currentNode = node + 1;
}

// 回跳到已完成节点（自定义生成允许）
function jumpToNode(node) {
  if (customProgress.completedNodes.includes(node)) {
    customProgress.currentNode = node;
  }
}
```

## 四、跨节点数据传递

```
节点1输出 → 节点2输入
  currentResult.images[selectedImageIndex].url → 去背景/编辑的image_url

节点2输出 → 节点3输入
  currentResult.processedImageUrl || images[selected].url → 视频生成的image_url

节点1+3输出 → 节点4输入
  currentResult.prompt → 收藏图片Prompt
  currentResult.video.script → 收藏视频脚本

节点4操作 → 跨模块
  切换平台 → currentResult + productInfo → 平台切换工作流
  开始新任务 → 重置所有状态
```
