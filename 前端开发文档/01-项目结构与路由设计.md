# 01 — 项目结构与路由设计

## 一、页面文件结构

```
coze-custom-page/
├── app.html                    # 应用入口
├── styles/
│   ├── variables.css           # CSS变量（色彩、间距、字号）
│   ├── global.css              # 全局样式、重置样式
│   └── responsive.css          # 响应式断点样式
├── components/
│   ├── TopNav/                 # 顶部导航栏
│   ├── EntryPanel/             # 左侧入口面板
│   ├── StatusBar/              # 底部状态栏
│   ├── UserMenu/               # 用户菜单下拉
│   ├── PlatformSelector/       # 平台选择器
│   ├── TemplateCard/           # 模板卡片
│   ├── KeywordTags/            # 关键词标签
│   ├── StepProgress/           # 步骤进度指示器
│   ├── ResultPreview/          # 结果预览
│   ├── Toast/                  # 全局提示
│   └── SkeletonLoader/         # 骨架屏
├── views/
│   ├── ChatView/               # 对话视图
│   ├── QuickGenView/           # 快速生成视图
│   ├── CustomGenView/          # 自定义生成视图
│   │   ├── ImageGenNode/       #   节点1：图片生成
│   │   ├── ImageEditNode/      #   节点2：图片处理
│   │   ├── VideoGenNode/       #   节点3：视频生成
│   │   └── SaveNode/           #   节点4：保存收藏
│   └── MaterialView/           # 素材库视图
├── store/
│   └── appState.js             # 全局状态管理
├── services/
│   └── cozeApi.js              # Coze API调用封装
└── utils/
    ├── platform.js             # 平台规范工具函数
    └── feedback.js             # 反馈工具函数
```

## 二、路由表

| 路径 | 视图 | 说明 | 需登录 |
|------|------|------|--------|
| `/` | 重定向到 `/quick` | 默认进入快速生成 | 是 |
| `/chat` | ChatView | 对话式交互 | 是 |
| `/quick` | QuickGenView | 快速生成4步表单 | 是 |
| `/custom` | CustomGenView | 自定义生成节点操作 | 是 |
| `/material` | MaterialView | 素材库 | 是 |
| `/settings` | SettingsView | 个人设置 | 是 |

> **注**：Coze 内置用户系统自动处理登录拦截，未登录用户访问时 Coze 自动弹出登录页，无需手动实现路由守卫。

## 四、TopNav 顶部导航栏

### 结构

```
┌─────────────────────────────────────────────────────────────┐
│  [Logo图标] 电商AI商图视频助手          [🛒平台▼] [📁] [👤▼] │
└─────────────────────────────────────────────────────────────┘
```

### 元素说明

| 元素 | 位置 | 交互 |
|------|------|------|
| Logo + 应用名 | 左侧 | 点击回到首页 `/quick` |
| 平台选择器 | 右侧 | 下拉选择6大平台，选中后全局生效 |
| 素材库按钮 | 右侧 | 点击跳转 `/material` |
| 用户头像 | 右侧 | 点击展开 UserMenu 下拉 |

### 平台选择器（导航栏模式）

导航栏中用精简的下拉选择器，快速生成Step1中用卡片式选择器（同组件两种模式）。

```html

  <option value="拼多多">🛒 拼多多 (750×750)</option>
  <option value="淘宝">🛒 淘宝 (800×800)</option>
  <option value="京东">🛒 京东 (800×800)</option>
  <option value="抖音">🛒 抖音 (1080×1080)</option>
  <option value="快手">🛒 快手 (750×750)</option>
  <option value="小红书">🛒 小红书 (1080×1440)</option>
</select>
```

### 入口项配置

| 入口 | 图标 | 路由 | 模式标识 |
|------|------|------|---------|

- 点击记录 → 加载对应的商品信息 + 结果
- 来源：调用素材查询工作流，传入 `user_id`

## 六、StatusBar 底部状态栏

```
┌─────────────────────────────────────────────────────────────┐
│  🛒 拼多多  |  🔄 生成图片中...  |  💎 75,320积分  |  进阶版   │
└─────────────────────────────────────────────────────────────┘
```

| 区域 | 内容 | 更新时机 |
|------|------|---------|
| 当前平台 | `currentPlatform` | 用户切换平台时 |
| 任务状态 | 当前执行中的任务描述 | 工作流调用开始/结束时 |
| 积分余额 | `auth.credits` | 生成操作后 |
| 套餐等级 | `auth.plan` | 登录后/升级后 |
