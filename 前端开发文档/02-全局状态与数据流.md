# 02 — 全局状态与数据流

## 一、AppState 定义

```typescript
interface AppState {
  // ── 认证状态 ──
  auth: {
    isLoggedIn: boolean;
    source: 'coze' | 'cloudflare' | 'custom';
    user: {
      id: string;              // sys.user_id
      name: string;            // sys.user_name
      avatar: string;
      phone?: string;
      email?: string;
    } | null;
    credits: number;           // 积分余额
    plan: 'free' | 'pro' | 'premium';
  };

  // ── 全局路由 ──
  currentPlatform: Platform;
  currentMode: 'chat' | 'quick' | 'custom' | 'material';

  // ── 商品信息（跨模块共享）──
  productInfo: {
    name: string;
    category: Category;
    sellingPoints: string[];
    keywords: string[];         // 已采纳的关键词
    sourcePlatform?: string;    // 链接提取时的来源平台
    sourceUrl?: string;         // 原始链接
    mainImageUrl?: string;      // 链接提取的主图
  };

  // ── 当前生成结果 ──
  currentResult: {
    images: Array<{
      url: string;
      prompt: string;
    }>;
    video: {
      url: string;
      script: string;
    } | null;
    selectedImageIndex: number;
    prompt: string;
    processedImageUrl?: string;
  };

  // ── 快速生成进度 ──
  quickProgress: {
    currentStep: 1 | 2 | 3 | 4;
    selectedStyle: string;
    selectedTemplate: string;
    selectedVideoTemplate: string | null;
  };

  // ── 自定义生成进度 ──
  customProgress: {
    currentNode: 1 | 2 | 3 | 4;
    completedNodes: number[];
    imageGenCompleted: boolean;
    imageEditCompleted: boolean;
    videoGenCompleted: boolean;
  };
}

// ── 枚举类型 ──
type Platform = '拼多多' | '淘宝' | '京东' | '抖音' | '快手' | '小红书';
type Category = '服饰' | '美妆' | '食品' | '家居' | '数码' | '母婴' | '其他';
type ImageStyle = '白底主图' | '场景图' | '模特图' | '促销图' | '详情图';
type VideoStyle = '展示型' | '对比型' | '开箱型' | '促销型';
```

## 二、Coze API 对接

### 2.1 工作流调用方式

在 Coze 自定义页面中，通过 **Coze SDK** 调用工作流：

```javascript
// 调用工作流示例
const result = await coze.runWorkflow({
  workflowId: 'xxx',        // 工作流ID
  parameters: {
    user_id: coze.getUserId(),  // 自动获取当前用户ID
    platform: '拼多多',
    category: '服饰',
    // ...其他参数
  }
});
```

### 2.2 Bot对话调用方式

```javascript
// 发送消息给Bot
const response = await coze.chat({
  botId: 'xxx',
  userId: coze.getUserId(),
  message: '帮我做一张拼多多运动鞋白底图',
});

// Bot自动进行意图识别，串联工作流
```

### 2.3 用户信息获取

```javascript
// Coze 内置用户系统
const userId = coze.getUserId();      // 对应 sys.user_id
const userName = coze.getUserName();  // 对应 sys.user_name

// 工作流中直接引用系统变量
// {{sys.user_id}}  → 自动替换为当前用户ID
// {{sys.user_name}} → 自动替换为当前用户昵称
```

## 三、工作流映射表

每个前端操作对应一个 Coze 工作流，所有工作流都需传入 `user_id`：

| # | 前端操作 | 工作流 | 输入参数 | 输出 |
|---|---------|--------|---------|------|
| 1 | 快速生成 → [一键生成] | 快速生成工作流 | user_id, platform, category, image_style, product_name, product_info, video_style | images[], video?, prompt |
| 2 | 自定义 → 图片生成 → [生成图片] | 图片生成工作流 | user_id, product_desc, category, platform, keywords, use_template | images[], prompt, recommend_keywords |
| 3 | 自定义 → 图片处理 → [去背景] | 去背景工作流 | user_id, image_url | processed_image_url |
| 4 | 自定义 → 图片处理 → [换背景/调尺寸/加水印] | 图片编辑工作流 | user_id, image_url, action, params, platform | processed_image_url |
| 5 | 自定义 → 视频生成 → [生成视频] | 视频生成工作流 | user_id, image_url, product_name, platform, video_desc, video_template | video_url, script |
| 6 | 保存收藏 → [收藏] | 模板收藏工作流 | user_id, prompt, template_name, category, platform, type | 收藏确认 |
| 7 | 素材库 → [查询] | 素材查询工作流 | user_id, query, platform | 素材列表 |
| 8 | 切换平台 → [确认切换] | 平台切换工作流 | user_id, current_prompt, product_info, target_platform | 新images[], 新prompt, 差异说明 |
| 9 | 粘贴链接 → [提取] | 链接提取工作流 | user_id, url | product_name, price, category, selling_points, main_image, platform |
| 10 | 关键词 → [查询] | 关键词补全工作流 | user_id, platform, category | keywords[] |

### user_id 传递规则

- **所有工作流**的开始节点必须包含 `user_id` 输入参数
- 值为 `{{sys.user_id}}`，Coze 自动替换为当前登录用户ID
- 素材库、收藏等按 `user_id` 实现数据隔离

## 四、数据流转图

```
┌───────────────────────────────────────────────────────────────┐
│                        前端数据流                               │
│                                                               │
│  用户输入                                                      │
│  ├── 快速生成表单 → 组装参数 → coze.runWorkflow → 结果写入    │
│  ├── 自定义节点操作 → 逐节点调用 → 逐步更新 currentResult     │
│  ├── 对话输入 → coze.chat → Bot意图识别 → 自动串联工作流      │
│  └── 链接输入 → 链接提取工作流 → 填充 productInfo             │
│                                                               │
│  数据流向                                                      │
│  ┌──────────┐     ┌──────────────┐     ┌──────────────┐       │
│  │ 用户输入  │ ──→ │ AppState     │ ──→ │ Coze API     │       │
│  │ 表单/对话 │     │ 全局状态     │     │ 工作流/Bot   │       │
│  └──────────┘     └──────┬───────┘     └──────┬───────┘       │
│                          │                    │               │
│                          │ ← 结果回写 ←──────│               │
│                          ▼                                    │
│                   ┌──────────────┐                            │
│                   │ 组件重渲染   │                            │
│                   │ ResultPreview│                            │
│                   │ Toast提示    │                            │
│                   └──────────────┘                            │
└───────────────────────────────────────────────────────────────┘
```

## 五、关键数据流转场景

### 5.1 快速生成

```
用户填写4步表单
    │
    ▼
组装参数：{ user_id, platform, category, image_style, product_name, product_info, video_style }
    │
    ▼
调用 coze.runWorkflow('快速生成工作流', params)
    │
    ▼
返回结果 → 写入 AppState.currentResult
    │
    ▼
ResultPreview 组件重渲染，展示3张图+视频
```

### 5.2 自定义生成（跨节点数据传递）

```
节点1: 图片生成
    │ 输入: productInfo → 图片生成工作流
    │ 输出: images[] → 写入 currentResult.images
    │       prompt → 写入 currentResult.prompt
    │ 用户选图 → currentResult.selectedImageIndex
    ▼
节点2: 图片处理
    │ 输入: currentResult.images[selectedImageIndex] → 图片编辑工作流
    │ 输出: processedImageUrl → 写入 currentResult.processedImageUrl
    ▼
节点3: 视频生成
    │ 输入: currentResult (选中的图片) + videoTemplate → 视频生成工作流
    │ 输出: video → 写入 currentResult.video
    ▼
节点4: 保存收藏
    │ 输入: currentResult.prompt + currentResult.video.script → 收藏工作流
    │ 输出: 收藏确认 → Toast提示
```

### 5.3 链接提取 → 填充表单

```
用户粘贴URL
    │
    ▼
调用链接提取工作流
    │
    ▼
返回: { product_name, price, category, selling_points, main_image, platform }
    │
    ▼
写入 AppState.productInfo
    │
    ▼
快速生成Step3表单自动填充
```

### 5.4 平台切换

```
用户点击"切换平台" → 选择目标平台 → 确认
    │
    ▼
读取 AppState.productInfo + currentResult.prompt
    │
    ▼
调用平台切换工作流( current_prompt, product_info, target_platform )
    │
    ▼
返回: 新images[], 新prompt, 差异说明
    │
    ▼
更新 currentResult + currentPlatform
    │
    ▼
ResultPreview 重渲染 + Toast展示差异说明
```
