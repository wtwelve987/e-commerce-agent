## 1. PlatformSelector 平台选择器

### 两种模式

| 模式 | 用途 | 样式 |
|------|------|------|
| `dropdown` | 导航栏全局切换 | 下拉选择框 |
| `card` | 快速生成Step1 | 6宫格卡片+平台规范摘要 |

### Card模式布局

```
┌─────────────────────────────────────────────┐
│  🛒 选择目标平台                             │
│                                             │
│  ┌────────┐ ┌────────┐ ┌────────┐          │
│  │  拼多多  │ │  淘宝  │ │  京东  │          │
│  │  选中✓  │ │       │ │       │          │
│  │ 750×750│ │800×800│ │800×800│          │
│  └────────┘ └────────┘ └────────┘          │
│  ┌────────┐ ┌────────┐ ┌────────┐          │
│  │  抖音   │ │  快手  │ │ 小红书 │          │
│  │1080×1080│ │750×750│ │1080×1440│         │
│  └────────┘ └────────┘ └────────┘          │
│                                             │
│  选中平台规范：                              │
│  · 主图: 750×750 · 视频: 1:1 · 风格: 简洁   │
```

### Props

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `mode` | `'card' \| 'dropdown'` | `'card'` | 展示模式 |
| `value` | `Platform` | `'拼多多'` | 当前选中 |
| `onChange` | `(platform: Platform) => void` | — | 切换回调 |
| `showSpec` | `boolean` | `true` | 是否显示规范摘要 |

### 交互规则

- 选中后即时高亮+勾选标记
- Card模式选中后下方展示该平台规范摘要
- 导航栏切换时：如有进行中的任务，弹窗确认

## 2. TemplateCard 模板卡片

### 布局

```
┌─────────────────┐
│  ┌───────────┐  │
│  │           │  │
│  │  预览图   │  │
│  │  128×128  │  │
│  │           │  │
│  └───────────┘  │
│  运动鞋白底图    │
│  📸图片 · 拼多多 │
│  750×750        │
└─────────────────┘
```

### Props

| 属性 | 类型 | 说明 |
|------|------|------|
| `id` | `string` | 模板ID |
| `name` | `string` | 模板名称 |
| `thumbnail` | `url` | 预览图 |
| `type` | `'image' \| 'video'` | 类型 |
| `platform` | `string` | 适配平台 |
| `size` | `string` | 输出尺寸 |
| `selected` | `boolean` | 是否选中 |
| `onSelect` | `(id: string) => void` | 选中回调 |

### 交互

- 点击选中：边框高亮 + 勾选标记
- 悬停：显示半透明"使用"按钮覆盖层
- 响应式：桌面3-4列 / 平板2-3列 / 手机横向滚动

## 3. KeywordTags 关键词标签

### 布局

```
┌─────────────────────────────────────────────┐
│  💡 推荐关键词                                │
│                                             │
│  [9.9] [秒杀] [性价比] [平替] [防摔] [透气] │
│                                             │
│  已采纳:                                    │
│  [9.9 ×] [性价比 ×]                        │
│                                             │
│  [全部采纳] [清空]                           │
└─────────────────────────────────────────────┘
```

### Props

| 属性 | 类型 | 说明 |
|------|------|------|
| `keywords` | `string[]` | 推荐关键词 |
| `selected` | `string[]` | 已采纳 |
| `onAdd` | `(keyword: string) => void` | 添加 |
| `onRemove` | `(keyword: string) => void` | 移除 |
| `onAddAll` | `() => void` | 全部采纳 |
| `onClear` | `() => void` | 清空 |

### 交互

- 点击推荐词 → 移入"已采纳"区，原标签变灰不可点击
- 点击已采纳词的 × → 移回推荐区
- 全部采纳：一键添加所有推荐词
- 数据来源：调用关键词补全工作流，传入 platform + category

## 4. StepProgress 步骤进度条

### 两种模式

**快速生成（线性，不可回跳）**：
```
  ①平台 ──── ②品类+风格 ──── ③商品信息 ──── ④视频 ──── 完成
   ●完成       ●当前          ○待完成       ○待完成     ○待完成
```

**自定义生成（可回跳已完成节点）**：
```
  ①图片生成 ──── ②图片处理 ──── ③视频生成 ──── ④保存收藏
   ●完成(可点)    ●当前         ○可选          ○可选
```

### Props

| 属性 | 类型 | 说明 |
|------|------|------|
| `steps` | `{label: string, status: 'completed' \| 'current' \| 'pending'}[]` | 步骤列表 |
| `mode` | `'quick' \| 'custom'` | 模式 |
| `onStepClick` | `(index: number) => void` | 点击回调(仅custom模式) |

## 6. Toast 全局提示

### 类型与样式

| 类型 | 颜色 | 图标 | 自动消失 |
|------|------|------|---------|
| `success` | 绿色 `#52c41a` | ✅ | 3秒 |
| `warning` | 橙色 `#faad14` | ⚠️ | 3秒 |
| `error` | 红色 `#ff4d4f` | ❌ | 5秒 |
| `info` | 蓝色 `#1890ff` | ℹ️ | 3秒 |

### Props

| 属性 | 类型 | 说明 |
|------|------|------|
| `type` | `'success' \| 'warning' \| 'error' \| 'info'` | 类型 |
| `message` | `string` | 提示内容 |
| `duration` | `number` | 显示时长(ms) |

### 位置：顶部居中，堆叠显示

## 7. SkeletonLoader 骨架屏

### 图片骨架

```css
.skeleton-image {
  width: 200px;
  height: 200px;
  background: linear-gradient(90deg, #f0f0f0 25%, #e0e0e0 50%, #f0f0f0 75%);
  background-size: 200% 100%;
  animation: shimmer 1.5s infinite;
  border-radius: 8px;
}

@keyframes shimmer {
  0% { background-position: 200% 0; }
  100% { background-position: -200% 0; }
}
```

### 视频骨架

```css
.skeleton-video {
  width: 100%;
  height: 240px;
  /* 同上shimmer动画 */
}
```

### 使用场景

| 场景 | 骨架数量 | 尺寸 |
|------|---------|------|
| 图片生成等待 | 3个 | 200×200 |
| 视频生成等待 | 1个 | 100%×240 |
| 模板列表加载 | N个 | 160×160 |
| 素材列表加载 | N个 | 120×120 |

## 8. UserMenu 用户菜单

### 布局

```
┌─────────────────────────────────┐
│  👤 张三                         │
│  zhang@example.com               │
│  ─────────────────────────────  │
│  💎 积分余额: 75,320            │
│  📦 套餐: 进阶版                 │
│  ─────────────────────────────  │
│  ⚙️ 个人设置                     │
│  📊 使用统计                     │
│  💳 充值/升级                    │
│  📝 使用记录                     │
│  ─────────────────────────────  │
│  🚪 退出登录                     │
└─────────────────────────────────┘
```

### Props

| 属性 | 类型 | 说明 |
|------|------|------|
| `user` | `UserInfo \| null` | 来自 `coze.getUserId()` / `coze.getUserName()` |
| `credits` | `number` | 积分余额 |
| `plan` | `string` | 套餐等级 |
| `onLogout` | `() => void` | 退出登录 |
| `onSettings` | `() => void` | 打开设置 |
| `onUpgrade` | `() => void` | 充值升级 |

### 数据来源

- `user.id` ← `coze.getUserId()` / `{{sys.user_id}}`
- `user.name` ← `coze.getUserName()` / `{{sys.user_name}}`
- `credits` / `plan` ← 通过工作流查询或Coze用户变量
