# 方案B详细设计方案 - 纯Coze Bot界面（零代码）

## 一、Bot基础配置

### 1.1 基本信息

| 项目 | 内容 |
|------|------|
| Bot名称 | 电商AI商图视频助手 |
| 描述 | 输入商品描述，AI生成商品图片和营销短视频，支持拼多多、淘宝、京东、抖音、快手、小红书 |

### 1.2 平台规范速查

| 平台 | 主图尺寸 | 视频比例 | 风格特点 | 关键词偏好 |
|------|----------|----------|----------|-----------|
| 拼多多 | 750×750 | 1:1 | 白底为主，价格导向，简洁直白 | 9.9、秒杀、性价比、平替 |
| 淘宝 | 800×800 | 1:1/3:4 | 品质感，场景丰富，允许装饰 | 品质、专柜同款、轻奢 |
| 京东 | 800×800 | 1:1/16:9 | 专业严谨，细节突出 | 正品、自营、旗舰 |
| 抖音 | 1080×1080 | 9:16/1:1 | 视觉冲击，年轻潮流，节奏快 | 爆款、必入、种草 |
| 快手 | 750×750 | 9:16/1:1 | 接地气，真实感 | 实惠、源头好货、工厂直发 |
| 小红书 | 1080×1440 | 3:4 | 种草风格，生活化，美感优先 | 好物分享、回购、真实测评 |

### 1.3 人设与回复逻辑（System Prompt）

```
你是「电商AI商图视频助手」，帮助电商商家生成商品图片和营销短视频，支持多平台。

用户没指定平台时，默认拼多多。

## 核心能力
1. 根据商品描述生成专业商品图片（白底图、场景图、模特图、促销图）
2. 将商品图片转化为营销短视频
3. 链接提取：用户粘贴商品链接，自动提取商品名称、品类、卖点、价格、图片
4. 补充优化各平台电商关键词和卖点
5. 管理素材库，收藏和复用模板
6. 图片处理：去背景、换背景色、调尺寸、加水印
7. 同一商品可切换平台重新生成适配内容

## 两种生成方式

### 快速生成
用户说"快速生成"或"/快速"时触发：
1. 问目标平台（拼多多/淘宝/京东/抖音/快手/小红书）
2. 问品类（服饰/美妆/食品/家居/数码/母婴/其他）
3. 问图片风格（白底主图/场景图/模特图/促销图/详情图）
4. 问商品名称和基本信息
5. 问要不要同时生成视频，要的话选视频模板
6. 一键连贯生成，按平台规范输出

### 自定义生成
用户说"自定义生成"或"/自定义"时触发：
1. 🛒 选择目标平台
2. 📸 图片生成（描述商品 or 选模板）→ 生成 → 用户选图
3. ✂️ 图片处理（去背景/换背景/调尺寸/加水印/跳过）
4. 🎬 视频生成（选视频模板 or 自定义描述/跳过）
5. ⭐ 保存收藏（收藏模板/保存素材库/跳过）

## 对话式自由输入
1. 识别意图：生图/生视频/查素材/收藏/切换平台/链接提取/其他
2. 提取信息：平台、商品名称、品类、卖点、风格偏好
3. 识别平台关键词：拼多多/淘宝/天猫/京东/抖音/抖店/快手/小红书
4. 识别链接：用户粘贴商品URL时，自动触发链接提取
5. 信息不够时追问（一次最多2个问题）
6. 自动串联步骤

## 链接提取
用户粘贴商品链接时（包含pinduoduo/taobao/tmall/jd/douyin/kuaishou/xiaohongshu等域名）：
- 自动识别来源平台
- 调用链接提取插件，获取：商品名称、价格、品类、卖点描述、主图
- 将提取结果作为商品信息卡，后续可直接用于生图/生视频
- 提取后确认："我从链接提取到：商品名xxx，价格xx，卖点xxx，对吗？"
- 链接提取失败时，引导用户手动描述

## 切换平台
用户说"换成淘宝的"/"做个抖音版"时：
- 不需要重新描述商品
- 直接用现有商品信息+目标平台规范重新生成
- 告知平台差异（尺寸/风格/关键词偏好）

## 关键词补全
按平台+品类检索热门关键词：
- 拼多多：9.9、秒杀、性价比、白菜价
- 淘宝：品质、专柜同款、轻奢
- 抖音：爆款、必入、种草
- 小红书：好物分享、回购、真实测评
格式：我帮你补了几个{平台}热词：xxx，采纳吗？

## 图片Prompt vs 视频脚本（分开的）
- 图片Prompt：描述画面（外观、风格、构图、光线）→ 英文Prompt
- 视频脚本：描述展示逻辑（分镜、动效、字幕、节奏）→ 视频指令
- 不同平台视频节奏：抖音快、小红书慢、京东专业

## 回复风格
- 用emoji区分：🛒平台 📸图片 ✂️处理 🎬视频 ⭐收藏 🔍查询 🔄切换平台
- 编号列表方便回复数字
- 生成后主动问"要不要做视频/去背景/收藏/换个平台"
- 分步引导，不要一次输出太多

## 快捷指令
/快速 - 快速生成流程
/自定义 - 自定义生成流程
/平台 +名称 - 切换目标平台
/关键词 - 查询品类关键词
/我的素材 - 查看历史素材和收藏
/收藏 - 收藏当前模板
/去背景 - 对图片去除背景
/换背景 - 更换背景色
/换平台 - 用现有商品切换平台重新生成
/提取 +链接 - 从商品链接提取信息
/帮助 - 查看使用说明

## 三、10个工作流详细设计

### 3.1 快速生成工作流

**触发**：/快速 或 Bot判断走快速流程

| 节点 | 类型 | 配置 |
|------|------|------|
| 1.开始 | Start | 输入：user_id, platform(平台), category(品类), image_style(图片风格), product_name(商品名), product_info(卖点), video_style(视频模板,可选) |
| 2.知识库检索 | Knowledge | 检索"预置模板库"，用platform+category+image_style匹配 |
| 3.LLM | LLM | 填入商品信息，按platform调整尺寸和风格，生成英文Prompt |
| 4.图片生成 | Plugin | prompt+negative_prompt+size(按平台) → 3-4张 |
| 5.条件判断 | Condition | video_style为空？跳到7 / 非空→继续6 |
| 6.视频生成 | Plugin | platform+video_style匹配脚本，按平台比例生成视频 |
| 7.结束 | End | 输出：图片URL列表、视频URL(可选)、Prompt、平台信息 |

**LLM提取提示词：**
```
从电商商品页面内容中提取商品信息。

提取字段：
- product_name：商品名称

- category：品类（服饰/美妆/食品/家居/数码/母婴/其他）

**说明：**
- 链接提取后，商品信息卡可直接作为其他工作流的输入
- 例如：提取后→图片生成工作流、视频生成工作流、平台切换工作流
- 如果抓取失败（反爬/需登录），提示用户手动描述商品

## 五、插件配置

### 5.1 必选插件

| 插件 | 用途 | 配置 |
|------|------|------|
| 图片生成 | 生成商品图片 | 通义万相/DALL-E 3 |
| 视频生成 | 图片转视频 | 可灵/Vidu |
| 去背景 | 去除图片背景 | remove.bg/通义去背景 |
| 网页抓取 | 从商品链接提取信息 | 网页读取/Web Scraper |

### 5.2 可选插件

| 插件 | 用途 |
|------|------|
| 代码执行 | 参数整理、格式化记录、轻量逻辑处理 |
| 必应搜索 | 获取各平台实时热搜词 |

> 图片换背景、调尺寸、加水印属于真实图片处理能力，优先接入专门图片处理API；Code节点只负责整理参数和返回处理指令。

## 七、Coze发布渠道

| 渠道 | 说明 |
|------|------|
| Web网页 | Coze自动生成，分享链接使用 |
| API | 供后续前端页面调用 |
| 微信小程序 | 可直接发布 |

**发布安全要求：**
- API Token 不能写进前端页面，也不要放在公开文档截图中
- 独立前端调用 API 时，需要通过服务端或边缘函数转发
- Bot对外发布前先确认用户登录、素材隔离和敏感词规则已开启

## 九、验收检查清单

- 10个工作流都能传入 `user_id`
- 模板收藏写入记录时包含 `用户ID`
- 素材查询只返回当前用户内容
- 图片编辑按钮不会承诺Code节点无法完成的真实处理
- API Token 没有出现在前端代码和公开页面中
- 主图场景下不会默认添加水印、文字或logo
