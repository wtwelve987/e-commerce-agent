# 拼多多AI图片视频生成应用 - 设计方案

## 一、项目定位

这是一个**完整应用项目**，不是单个工作流/Agent。包含页面、对话、多种入口和核心能力。

---

## 二、两大入口

### 入口1：对话式
- 页面内聊天框，用户自由说话
- Bot做意图识别，自动串联流水线
- 适合不知道怎么选、想快速描述需求的用户

### 入口2：选择式
- 页面操作面板，点击选择
- 分"快速生成"和"自定义生成"两条路径
- 适合明确知道要什么的用户

---

## 三、快速生成 vs 自定义生成

| | 快速生成 | 自定义生成 |
|--|---------|-----------|
| 操作 | 选品类→选模板→填信息→(选视频模板)→一键出 | 逐节点操作，每步做完选下一步 |
| 图片 | 预置模板直接出图 | 可描述+可选模板 |
| 视频 | 选模板后自动连贯生成 | 单独一步，可选模板或自定义描述 |
| 图片处理 | 不含 | 可选去背景/换背景/调尺寸/加水印 |
| 适合 | 日常高频、标准商品 | 特殊需求、精细控制 |
| 速度 | 快(1次等待) | 慢(多步但精准) |

---

## 四、自定义生成节点详解

自定义生成是逐步操作，每个节点用户自主决定：

```
节点1: 图片生成
├── 输入商品描述(自由文本)
├── 或选择预置图片模板
├── 可补充关键词(知识库推荐)
└── 点击"生成图片" → 出3-4张

节点2: 图片处理(可选，可跳过)
├── 去除背景
├── 更换背景色
├── 调整尺寸
├── 添加水印
└── 跳过

节点3: 视频生成(可选，可跳过)
├── 选择预置视频模板
├── 或自定义视频描述
└── 点击"生成视频"

节点4: 保存/收藏
├── 下载图片/视频
├── 收藏Prompt为模板
├── 收藏脚本为模板
└── 保存到素材库
```

---

## 五、核心能力(8条流水线)

| 编号 | 流水线 | 作用 |
|------|--------|------|
| A | 商品信息采集 | 提取商品名/品类/卖点→结构化商品卡 |
| B | 图片生成 | 预置模板或描述→生图API→3-4张候选 |
| C | 视频生成 | 预置脚本或自定义描述→视频API→5-15s |
| D | 关键词补全 | 知识库检索→推荐热词/卖点词 |
| E | 模板收藏 | 保存满意Prompt/脚本→写入知识库 |
| F | 素材查询 | 查历史素材和收藏模板 |
| G | 去背景 | 调用去背景API→输出透明底图 |
| H | 图片编辑 | 换背景/调尺寸/加水印 |

---

## 六、知识库设计（4个库）

| 知识库 | 内容 | 用途 |
|--------|------|------|
| 关键词库 | 各品类热搜词、卖点词、场景词 | 补充用户描述 |
| 预置模板库 | 图片Prompt模板 + 视频脚本模板 | 快速生成和模板选择 |
| 文案模板库 | 标题、卖点文案、字幕模板 | 生成营销文案 |
| 我的素材库 | 历史生成记录 + 收藏模板 | 素材查询和复用 |

---

## 七、图片Prompt vs 视频脚本（分开的）

```
图片Prompt → 描述"画面长什么样"
├── 主体、风格、构图、光线、细节
└── 示例："Product photography, white sneakers, pure white bg, 8k..."

视频脚本 → 描述"怎么展示、什么节奏"
├── 分镜、动效、字幕、转场、时长
└── 示例："0-3s全景慢放→3-5s侧面滑动→5-7s底部特写→7-8s促销弹出"
```

## 八、技术架构

### 整体架构

```
┌──────────────────────────────────────────────────┐
│                   前端（用户界面）                  │
│                                                  │
│   对话界面 + 操作面板(快速/自定义选择)              │
└────────────────────┬─────────────────────────────┘
                     │ 调用
                     ▼
┌──────────────────────────────────────────────────┐
│              Coze平台（AI能力 + 后台）              │
│                                                  │
│  Bot(意图识别) + 工作流(8条流水线)                 │
│  + 知识库(4个) + 插件(生图/视频/去背景)             │
└──────────────────────────────────────────────────┘
```

### 关键结论：不需要单独的后台应用

| 层面 | 需要？ | 说明 |
|------|--------|------|
| AI侧 | ✅ 需要 | Coze的Bot+工作流就是AI侧，配置即可不需要开发 |
| 前端 | ✅ 需要 | 操作界面，可放在Coze里或独立部署 |
| 后台应用 | ❌ 不需要 | Coze本身就是后台，工作流+知识库+插件全有了 |

### 前端部署方案

| 方案 | 做法 | 开发量 | 自由度 |
|------|------|--------|--------|
| **A. Coze Vibe Coding** | 用自然语言让Coze生成前端页面，Coze托管部署 | 最低 | 中 |
| **B. Coze Bot界面** | 直接用Bot对话+快捷指令+卡片按钮，不做单独页面 | 零 | 低 |
| **C. 独立前端+Coze SDK** | 自己写前端页面，嵌入Coze Bot能力，部署到Vercel等 | 中 | 最高 |

### 推荐路径

```
第一步：方案B（零代码，1-2天）
│  纯Coze Bot界面，对话+快捷指令+卡片按钮
│  快速验证需求，看效果够不够用
│
第二步：方案A或C（按需升级）
│  如果Bot界面不够用：
│  · 想快速出页面 → 方案A（Vibe Coding生成+Coze托管）
│  · 想完全自定义 → 方案C（自己写前端+Coze SDK/API）
│
第三步：独立部署（可选）
│  前端部署到 Vercel / CloudBase / 自有服务器
│  后台仍然是Coze，不用自己写
```

### Coze中各功能的实现方式

| 功能 | Coze中的实现 |
|------|-------------|
| 对话入口 | Bot本身，人设里写意图识别规则 |
| 选择入口 | 快捷指令 / 自定义页面(Vibe Coding) |
| 快速生成 | 一个工作流：选品类→选模板→填信息→连贯执行 |
| 自定义生成-图片 | 一个工作流：描述/选模板→生图 |
| 自定义生成-图片处理 | 一个工作流：去背景/换背景/调尺寸 |
| 自定义生成-视频 | 一个工作流：图片+模板/描述→生视频 |
| 收藏 | 一个工作流：保存Prompt/脚本到知识库 |
| 素材查询 | 一个工作流：检索知识库 |
| 前端页面 | Vibe Coding生成 或 Web SDK嵌入 |

## 九、内置工具

| 工具 | 用途 | 实现方式 |
|------|------|----------|
| 去背景 | 去除图片背景 | 调用去背景API(如remove.bg) |
| 换背景 | 替换背景色/图 | 图片编辑API |
| 调尺寸 | 调整为拼多多规范尺寸 | 图片处理代码 |
| 加水印 | 添加品牌/店铺水印 | 图片处理代码 |

## 十、推荐API
